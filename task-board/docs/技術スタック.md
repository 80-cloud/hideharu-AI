## 1. フルスタック構成の全体像

### アプリ全体の構造

```
┌─────────────────────────────────────┐
│         ブラウザ（ユーザーの画面）         │
│                                     │
│   React + Tailwind CSS（フロントエンド）   │
│   ・タスクの表示・操作画面                 │
└──────────────┬──────────────────────┘
               │  HTTP通信（Axios）
               ▼
┌─────────────────────────────────────┐
│     Spring Boot サーバー（バックエンド）    │
│                                     │
│   Java + Gradle で動くWebサーバー        │
│   ・ブラウザからのリクエストを受け取る         │
│   ・タスクの追加・更新・削除の処理をする       │
└──────────────┬──────────────────────┘
               │  JPA/Hibernate（SQL自動生成）
               ▼
┌─────────────────────────────────────┐
│       PostgreSQL（データベース）          │
│                                     │
│   タスクのデータを永続的に保存               │
│   Docker コンテナ内で動作                │
└─────────────────────────────────────┘
```

### APIエンドポイント設計

フロントエンドとバックエンドは REST API で通信します。

| HTTPメソッド | URL | 操作 | v1.0の対応関数 |
|---|---|---|---|
| GET | `/api/tasks` | タスク一覧を取得 | `loadTasks()` |
| POST | `/api/tasks` | タスクを新規作成 | `addTask()` |
| PUT | `/api/tasks/{id}` | タスクを更新（編集・カラム移動） | `updateTask()` |
| DELETE | `/api/tasks/{id}` | タスクを削除 | `deleteTask()` |

## 4. インフラ・環境設定

### 4-1. Docker + docker-compose（開発環境のDB起動）

**`docker-compose.yml` の構成：**
```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_DB: taskboard
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    ports:
      - "5432:5432"
```

---

### 4-2. .env ファイル（環境変数管理）

パスワードやURLなどの機密情報をコードに直書きせず、`.gitignore` で除外管理します。

**バックエンド（Spring Boot）：**
```properties
# application.properties
spring.datasource.url=jdbc:postgresql://localhost:5432/taskboard
spring.datasource.username=${DB_USER}
spring.datasource.password=${DB_PASSWORD}
```

**フロントエンド（React + Vite）：**
```
# .env
VITE_API_BASE_URL=http://localhost:8080
```
```js
const API_URL = import.meta.env.VITE_API_BASE_URL;
```

## 6. 不採用技術とその理由

| 技術名 | 不採用の理由 |
|---|---|
| Next.js | React単体で十分。SSR（サーバーサイドレンダリング）が必要な場合に採用を検討する |
| TypeScript | 今回は学習コスト削減のためJavaScriptを使用。将来の拡張時に移行を検討する |
| Redux / Zustand | このアプリの規模には過剰。React標準のuseContextで十分 |
| Create React App | 2023年に公式で非推奨となったため不採用 |
| Liquibase | FlywayよりSQLの書き方が複雑 |
| MySQL / SQLite | PostgreSQLを選定。型が豊富で本番環境でも広く使われている |
| localStorage | サーバーへの移行により不要 |

## 7. 外部ライブラリ・依存関係

### バックエンド（build.gradle に記述した依存関係）

| ライブラリ名 | 役割 |
|---|---|
| spring-boot-starter-web | HTTPリクエストを受け取るWebサーバー機能 |
| spring-boot-starter-data-jpa | データベース操作（JPA/Hibernate） |
| spring-boot-starter-actuator | ヘルスチェックエンドポイント |
| postgresql | JavaとPostgreSQLをつなぐドライバー |
| flyway-core | データベースのバージョン管理 |
| flyway-database-postgresql | Flyway の PostgreSQL 対応モジュール |
| lombok | `@Getter`/`@Setter` などの定型コードを自動生成 |

### フロントエンド（npm でインストールするライブラリ）

| ライブラリ名 | 役割 |
|---|---|
| react, react-dom | Reactの本体（v19.2.5） |
| axios | APIとの HTTP通信（v1.15.2） |
| @tailwindcss/vite | Tailwind CSS v4 の Vite プラグイン（v4.2.4） |
| @vitejs/plugin-react | ViteでReactを使うためのプラグイン |

> **補足：** Tailwind CSS v4 では従来の `postcss` + `autoprefixer` は不要になり、`@tailwindcss/vite` プラグイン1つで完結します。
