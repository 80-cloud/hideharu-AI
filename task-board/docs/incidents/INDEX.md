# Incident Index — 異常検知・診断ライブラリ

> 過去のインシデント記録。**新しい異常に直面した時は最初にここを開く。**
> 単純な1対1比較ではなく、**症状 → 兆候 → 真因** の3段階で照合すること。

---

## 使い方

1. **症状をキーワードで `grep`** → cases/*.md の Symptoms セクションを検索
2. ヒットしたケースの **「兆候（Early Signs）」** が現状と一致するか確認
3. 一致 → 既存の **「是正処置」** を参考に対処
4. 不一致 → 新規ケースとして記録対象（解決後に追加）

## メンテナンスルール

- 新規ケース：`cases/YYYY-MM-DD-短縮タイトル.md` 形式
- ケースには **症状 / 兆候 / 真因 / 関連パターン / 是正処置 / 再発防止** を必ず記載
- ステータス：`Open` / `Resolved` / `Reopened` のいずれか
- 解決後 1週間以内に「再発防止」セクションを更新（Jidoka の検証）
