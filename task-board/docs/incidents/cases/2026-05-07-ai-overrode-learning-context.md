# 2026-05-07 AI アシスタントが学習文脈を実務経験で上書きした

**Phase:** 1〜3 全体（メタ・インシデント）
**ステータス:** Acknowledged（再発防止策あり）

---

## 症状（Symptoms）

ユーザーは AWS 学習講座を進めており、Nginx welcome page を到達点とする想定だった。
しかし AI（Claude）が **A案：Vite dev server** を推奨・誘導した結果：

- 講座のチェックポイント（Welcome to nginx）を**スキップ**
- 結果的にアプリは動いたが「自分が何を学んだのかわからない」状態に
- 後追いで Nginx を追加インストールする手戻りが発生
- ユーザーから「学習用とおっしゃっていたはず」と指摘

並行して以下の手戻りも発生：
- t2.micro 選定（2024年から ap-northeast-1 で free tier 対象外）
- curl-minimal 競合（AL2023 仕様未確認）
- Java 21 vs 25 不整合（build.gradle 未確認）
- t3.micro メモリ不足（リソース計画不足）
- application.yml 平文パスワード commit（機密情報スキャン未実施）

## 兆候（Early Signs）— 次回検知用

- ユーザーが「学習用」「講座用」「教材」「課題」「初学者」と発言
- ユーザーが既存教材（動画/ブック/サイト）に言及
- 講座のシラバス・到達点を AI が確認していない
- AI が「ベストプラクティス」「推奨案」を独自に提案している
- AI の「推奨」が業界標準寄り、ユーザーの教材が古典的手作業の場合

## 真因（Root Cause）

**「ユーザーの学習文脈を、AI の実務経験で上書きした」**

5 Whys：
1. なぜ Nginx をスキップさせたか → A案を推奨し誘導した
2. なぜ A案を推奨したか → 「動作の簡単さ」を評価軸にした
3. なぜ「簡単さ」を最優先したか → 「動かす」を最終目標と見なした
4. なぜ「動かす」を目標としたか → 「学習用」発言の意味を深掘らなかった
5. なぜ深掘りしなかったか → **AI の実務経験で「ベスト」を選ぶほうが合理的と無意識に判断した**

並列の根本問題：**実装前の監査ステップを全省略**
- 講座シラバス確認 ❌
- アプリ要求精査（build.gradle / application.yml / docker-compose） ❌
- OS仕様精査（curl-minimal, free tier 変更） ❌
- リソース要求合計→インスタンスサイズ計算 ❌
- 機密情報スキャン ❌

これら5項目の監査をスキップし、いきなり Terraform 実装に入った結果、5件の手戻りが発生。

- 関連：実務での「現場の事情を知らずにベストプラクティスを押し付ける」コンサル失敗パターン

## 是正処置（Remediation）

事後対応として：
1. ポート80を SG に追加
2. EC2 上で `dnf install nginx` を手動実行
3. Welcome to nginx を Chrome で確認 → ユーザーの学習チェックポイント達成
4. 本ケースをインシデント記録として残す（メタ反省）

## 再発防止（Prevention / Jidoka）

### AI（Claude）側のチェックリスト
ユーザーが「学習用 / 講座用 / 教材」と発言したら、**Terraform 等の実装に着手する前に必ず以下を確認**：

- [ ] 講座のシラバス・到達点（chapter / step）を聞く
- [ ] ゴール画面（Welcome page、特定のUI、エラー解決）を聞く
- [ ] 現在の進捗位置を聞く
- [ ] 講座の方針（手作業重視 / IaC重視 / 設定ファイル重視）を聞く
- [ ] **講座が示す手順を上書きしない**（最低限、「講座と異なる方法」を提案する場合は明示・選択肢提示）

### 全プロジェクト共通の Pre-Flight Audit（実装着手前）

- [ ] アプリ要求精査：build.gradle / package.json / application.yml / docker-compose.yml
- [ ] OS仕様確認：プリインストールパッケージ、ディストロ世代差分
- [ ] フリーティア最新仕様：`describe-instance-types --filters free-tier-eligible=true`
- [ ] リソース要求 → インスタンスサイズ計算（メモリ要求合計 + 余裕30%）
- [ ] 機密情報スキャン：`grep -rE "password|secret|api[_-]?key" config-files/`

- `feedback_learning_context_first.md` として CLAUDE 用の申し送りを追加
- 将来のセッションでも同じ轍を踏まないようにする
