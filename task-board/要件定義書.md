# 要件定義書

**プロジェクト名：** タスク管理ボード  
**作成日：** 2026年4月23日  
**バージョン：** 1.0  

---

## 1. プロジェクト概要

### 1-1. 背景・目的

個人のタスクを視覚的に管理するためのWebアプリケーションを開発する。  
Trello（タスク管理サービス）を参考に、シンプルで使いやすい画面構成を目指す。

### 1-2. 対象ユーザー

- 個人でタスクを管理したい人
- 技術知識がなくても直感的に操作できることを前提とする

### 1-3. 利用環境

| 項目 | 内容 |
|------|------|
| 対応ブラウザ | Google Chrome（最新版）、Safari（最新版） |
| デバイス | PC（スマホ対応は対象外） |
| インターネット接続 | 不要（オフラインで動作） |
| サーバー | 不要（ブラウザのみで動作） |

---

## 2. 機能要件

> 「機能要件」とは「システムが必ず持っていなければならない機能」のこと。

### 2-1. ボード画面

| No. | 機能 | 詳細 |
|-----|------|------|
| F-01 | 今日の日付表示 | 画面上部に「YYYY年M月D日（曜日）」形式で表示する |
| F-02 | カラム表示 | 「やること」「進行中」「完了」の3列を横並びで表示する |
| F-03 | タスク件数表示 | 各カラムのタスク数をバッジで表示する |

### 2-2. タスクカード

| No. | 機能 | 詳細 |
|-----|------|------|
| F-04 | タイトル表示 | タスクのタイトルを表示する（必須） |
| F-05 | 説明文表示 | タスクの説明文を表示する（任意） |
| F-06 | 優先度表示 | 「高／中／低」をカラーバッジで表示する |
| F-07 | 期限表示 | 期限日を「M/D」形式で表示する（任意） |
| F-08 | 期限切れ警告 | 期限を過ぎたカードに警告マーク（⚠）を赤色で表示する |

### 2-3. タスク操作

| No. | 機能 | 詳細 |
|-----|------|------|
| F-09 | タスク追加 | 各カラムの「＋ タスクを追加」ボタンからフォームを開いて追加できる |
| F-10 | タスク編集 | カードの「編集」ボタンから内容を変更できる |
| F-11 | タスク削除 | カードの「削除」ボタンから削除できる（確認ダイアログあり） |
| F-12 | カラム移動 | カードをドラッグ＆ドロップで別のカラムに移動できる |

### 2-4. 入力フォーム

| No. | 項目名 | 種別 | 必須 | 備考 |
|-----|--------|------|------|------|
| F-13 | タイトル | テキスト入力 | ○ | 最大文字数の制限なし |
| F-14 | 説明文 | テキストエリア | — | 任意入力 |
| F-15 | 優先度 | ドロップダウン | ○ | 高 / 中（初期値） / 低 |
| F-16 | 期限 | 日付入力 | — | カレンダーから選択 |

### 2-5. データ保存

| No. | 機能 | 詳細 |
|-----|------|------|
| F-17 | データ永続化 | ブラウザの localStorage にデータを保存する |
| F-18 | 自動読み込み | ページを開いたとき、保存済みのタスクを自動で表示する |

---

## 3. 非機能要件

> 「非機能要件」とは「動作の速さ・使いやすさ・安全性」など機能以外の品質に関する要件のこと。

| No. | 種別 | 内容 |
|-----|------|------|
| N-01 | パフォーマンス | ページ読み込みから1秒以内に表示が完了すること |
| N-02 | 操作性 | ボタンや入力欄は直感的に操作できること |
| N-03 | セキュリティ | ユーザーが入力した文字列はXSS対策（エスケープ処理）を施すこと |
| N-04 | 保守性 | 列の追加・優先度の変更などが設定値の編集だけでできること |
| N-05 | 可読性 | コードに日本語コメントを付け、初心者でも読めること |

---

## 4. 対象外（スコープ外）

以下は今回のバージョンでは開発しない。

- ユーザー認証（ログイン機能）
- 担当者の設定
- 複数人での共有・リアルタイム同期
- スマートフォン対応
- サーバーへのデータ保存

## 5. 画面設計

### 5-1. 画面一覧

| 画面名 | 説明 |
|--------|------|
| ボード画面 | メイン画面。カラムとカードを表示する |
| タスクフォーム（モーダル） | タスクの追加・編集を行うポップアップ画面 |

### 5-2. ボード画面 レイアウト概要

```
┌─────────────────────────────────────────────┐
│  タスク管理ボード          2026年4月23日（木）  │  ← ヘッダー
├──────────┬──────────┬──────────────────────┤
│ やること  │  進行中  │       完了            │
│  [3]     │  [1]    │       [2]            │  ← カラムヘッダー
│          │         │                      │
│ ┌──────┐ │ ┌──────┐│ ┌──────┐ ┌──────┐   │
│ │カード │ │ │カード ││ │カード │ │カード │   │  ← タスクカード
│ └──────┘ │ └──────┘│ └──────┘ └──────┘   │
│          │         │                      │
│ ＋追加   │ ＋追加  │ ＋追加               │  ← 追加ボタン
└──────────┴──────────┴──────────────────────┘
```

### 5-3. タスクカード 項目レイアウト

```
┌─────────────────────────────┐
│ タイトル（太字）              │
│ 説明文（グレー文字）           │
│ [高] [中] [低]   期限：4/30  │  ← 優先度バッジ・期限
│ ─────────────────────────── │
│  編集   削除                 │  ← 操作ボタン
└─────────────────────────────┘
```

## 6. データ設計

### 6-1. タスクデータの構造

ブラウザの localStorage に以下の形式でデータを保存する。

```json
[
  {
    "id":        "一意のID（自動生成）",
    "title":     "タスクのタイトル",
    "desc":      "説明文（空でも可）",
    "priority":  "high / medium / low",
    "dueDate":   "2026-04-30（空でも可）",
    "status":    "todo / doing / done",
    "createdAt": "2026-04-23T10:00:00.000Z"
  }
]
```

### 6-2. ステータスと表示カラムの対応

| status値 | 表示カラム |
|----------|----------|
| `todo`   | やること |
| `doing`  | 進行中 |
| `done`   | 完了 |

## 9. 用語集

| 用語 | 説明 |
|------|------|
| カラム | タスクを並べる縦の列（やること・進行中・完了） |
| カード | 1つのタスクを表示するブロック |
| モーダル | 画面の上に重ねて表示されるポップアップウィンドウ |
| localStorage | ブラウザに内蔵されているデータ保存の仕組み |
| XSS | 悪意のある文字列を埋め込む攻撃手法。対策としてエスケープ処理を行う |
| ドラッグ＆ドロップ | 要素をつかんで別の場所に移動させる操作 |
